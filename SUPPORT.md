# Support

Use GitHub issues for public questions, bug reports, and feature requests.

Before opening an issue:

- Check the README and docs under `docs/`.
- Search existing issues for the same backend, ESO version, or Helm chart value.
- Reduce the example to the smallest redacted `SecretStore` and
  `ExternalSecret` that reproduces the behavior.

Do not post real secrets, master passwords, API tokens, vault item IDs, vault
item names, Kubernetes Secret values, kubeconfigs, or private hostnames in
public issues.

Use GitHub Security Advisories for vulnerabilities or reports that require
private details.
