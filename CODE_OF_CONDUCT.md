# Code Of Conduct

This project follows a simple rule: keep technical collaboration respectful,
specific, and safe for public participation.

## Expected Behavior

- Be direct about technical risks while staying respectful toward people.
- Assume good intent, especially when reviewing first-time contributions.
- Keep disagreement focused on reproducible behavior, documented tradeoffs, and
  maintainable code.
- Help keep credentials, private infrastructure details, and personal data out
  of public issues, pull requests, logs, screenshots, and examples.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing another person's private information.
- Pressuring maintainers or contributors to disclose sensitive operational
  details.
- Repeatedly ignoring maintainer requests to keep discussions scoped and safe.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests;
or block participants when behavior makes collaboration unsafe or unproductive.

Report conduct concerns privately through GitHub Security Advisories if the
report includes sensitive information. Otherwise, contact the maintainers
through the repository's normal issue or pull request channels.
